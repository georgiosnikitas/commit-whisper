### Assumption-resolution pass (John, George, 2026-06-07)
George reviewed and resolved several `[ASSUMPTION]`/`[NOTE FOR PM]` items into firm decisions in `prd.md`:
- **§2.2** renamed "Non-Users" → **"Non-Target Users (v1)."**
- **FR-1 (branches):** retrieves all branches reachable without extra credentials **by default**; user may scope to a single named branch. (Assumption removed.)
- **FR-2 / FR-15 (secrets):** token + LLM key come **only from env vars or config file, never as CLI arguments**, in both interactive (shell-history safety) and CI (secret-injection) modes; only non-secret config (repo URL, provider, model, output formats) may be passed as flags.
- **FR-3 (faults):** **no retries/backoff.** On a transient or rate-limit provider error, fail fast — report clearly (provider + reset guidance where available), exit without a partial Report. commit-sage does not absorb faults outside its responsibility; re-running is the user's choice.
- **Retrieval mechanism:** lean to **`git clone` over REST/GraphQL API paging** (reduces rate-limit exposure, aligns with no-retry; cheap local diffs for Group E). Override only if a desired feature is technically unachievable via clone. Recorded as advisory leaning in `addendum.md`; FR-1 `[NOTE FOR PM]` updated to reference it.
- **§4.1 / §6.2 Out of Scope (confirmed v1 decisions):** self-hosted / generic git servers beyond GitHub/GitLab/Bitbucket — **deferred to v2**; **SSH-key auth — token-based only in v1.** Both `[ASSUMPTION]`/`[NOTE FOR PM]` tags removed; matching §13 Assumptions Index entry cleared.

Distinct from its static one-line description and from the repo-level three-part Narrative.
