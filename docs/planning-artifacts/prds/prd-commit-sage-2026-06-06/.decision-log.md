### Interaction-model & secret-sourcing pass (George, 2026-06-12)
George locked two decisions during the architecture workflow (recorded in `prd.md` §12 as #11 and #12):
- **Interaction model (#11):** zero arguments in an interactive terminal = interactive mode (a discoverable top-level menu plus guided prompts); one or more arguments, in any context = strict single-shot that hard-fails on any missing required input (typed error + machine-readable exit code) and never prompts; zero arguments in a non-interactive context (no interactive terminal / CI) fails fast the same way. The bare `commit-sage` command is the product's only interactive entry point; it never hangs waiting for input and never silently guesses. (Affects FR-14, FR-15, §3 Glossary.)
- **Secret sourcing (#12):** the git access token and the LLM key are read from **environment variables only** — never the config file, never a CLI flag, never an interactive prompt. Both secrets are handled identically; a missing secret is surfaced by naming its exact environment variable (argument mode hard-fails; the bare interactive command names it in the menu). This **supersedes** the 2026-06-07 bullet that allowed config-file secrets and retires the masked-prompt-and-save interactive behavior. (Affects FR-2, FR-11, FR-14, FR-15, §3 Glossary.)
