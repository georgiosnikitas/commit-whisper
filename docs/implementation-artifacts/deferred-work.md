- **`shared/ui.ts` will collide with the `no-console` rule** — the ESLint config applies `no-console: error` to `src/shared/**`, but the architecture designates `src/shared/ui.ts` as the single human-output writer to stderr. Minor; consider a dedicated config-typecheck pass (e.g. a second tsconfig) in a later tooling/CI story.

## Deferred from: code review of 1-2-two-phase-configuration-resolver-and-frozen-runconfig (2026-06-13)

- **Env value validation/normalization bundle** [src/config/env.ts] — decimal + safe-integer `maxCommits` (reject hex `0x10`, exponent `1e3`, and `> Number.MAX_SAFE_INTEGER` precision loss); case-insensitive enum parsing for `aiMode`/`provider`/`branch`/`format` (today `COMMIT_SAGE_AI_MODE=OFF` is silently ignored → can leave AI on in interactive `auto`); IANA `timezone` validation; ISO date validation + `startDate ≤ endDate`; `llmBaseUrl` URL validation (SSRF-relevant once `narrate` issues requests in Story 1.6); `outputFormats` dedup (`html,html` → double render), reject-invalid, and enforce ≥1. Deferred because the Story 1.2 Dev Notes explicitly scope value validation to the Zod schema story; 1.2's AC bar is pure merge + provenance + capability gate + typed gap-error, and `env.ts` is documented to coerce leniently and omit invalid values.
- **Treat JSON `null` as "not supplied"** [src/config/resolver.ts, src/config/gaps.ts] — `mergeLayers` uses `value !== undefined` and `finalizeRunConfig` checks `=== undefined`, so a `null` from a real config file would override a lower layer and/or pass the required-field check. Not reachable in 1.2 (the `PartialRunConfig` types forbid `null` and `configFile` is injected `{}`); address when Epic 6 wires the `~/.commit-sage` JSON reader.
- **Broaden + normalize CI detection** [src/config/capability.ts] — `detectCI` consults only `env.CI` and matches `"false"`/`"0"` case-sensitively, so `CI=False`/`CI=FALSE`/`CI=" "` are treated as CI (→ headless, the safe direction) while a deliberate `CI=False` is not honored as "not CI." Consider `CONTINUOUS_INTEGRATION` + provider-specific markers (or `ci-info`) and trim+lowercase the value. Deferred: current behavior fails closed; Task 4 listed broader markers only as a nice-to-have.
- **Single-source-of-truth for the `provider`/`format`/`aiMode` enums** [src/config/env.ts, src/config/run-config.ts] — the runtime `Set<string>` validators in `env.ts` are decoupled from the union types in `run-config.ts`, so adding/renaming a union member without updating the set compiles cleanly but mis-parses. Derive both the union and the runtime set from one `as const satisfies readonly Provider[]` array. Fold into the Zod schema story (the schema becomes the canonical enum source).
- **Enforce `entitlement.commitCap` against `maxCommits`** [src/config/resolve-run-config.ts] — the resolver carries `entitlement` but never clamps `maxCommits` to the tier cap. By design: the Free 100-commit cap mechanics (date-then-cap ordering, truncation notice) are Epic 2.
- **Replace the `as RunConfig` cast with an explicit mandatory-field invariant** [src/config/gaps.ts] — `FIELD_SPECS` marks the defaulted-mandatory fields (`branch`, `timezone`, `noMerges`, `outputFormats`, `aiMode`) `optional`, so `finalizeRunConfig` does not validate them and the `... as RunConfig` cast would emit a frozen object with `undefined` "required" fields if ever called without the defaults layer merged in. Safe today (the sole caller always merges `buildDefaults`, and the precondition is documented in the file header + exercised only by tests that build full partials); harden later by asserting the mandatory fields are present instead of casting.
